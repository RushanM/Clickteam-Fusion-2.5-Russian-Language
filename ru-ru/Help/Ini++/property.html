Панель свойств

Ниже приведён список опций в панели свойств для объекта INI++.

  • Если опция «Глобальные данные (Global Data)» отмечена, то этот Ini++ объект использует хранилище глобальных данных(global data store). Название хранилища глобальных данных задается в 'Ключ глобальных данных (Global data key)'. Все Ini++ объекты которые используют общий ключ данных используют и общие данные.
  • «Файл по умолчанию (Default File)» определяет путь файла для загрузки в начале сцены. Это может быть относительный путь. Если Выбрано 'Только чтение (Read Only)' то этот файл будет загружен в режиме только для чтения.
  • «Основные папки (Base Folders)» определяет путь который относителен путям, относительно к. Это может быть:
  • Windows Folder (Папка Windows)(например C:\Windows)
  • (Исходная) текущая папка. Это может быть изменено пользователем при запуске программы (например, если сделан кратчайший путь к программе его можно указать здесь). Если текущая папка изменяется в течение сцены, этот путь не будет изменён.
  • Папка в которой находится EXE. Когда программа запускается из MMF, это будет вместо папки в которую сохраняется файл MFA.
  • User Application Data (например, C:\Documents and Settings\Jax\Application Data)
  • Common Application Data (например, C:\Documents and Settings\All Users\Application Data)
    • Обратите внимание, что в ранних версиях Windows, последние две папки не существует. В этом случае будет использоваться каталог Windows.
    • 'Начальные данные (Initial Data)' текст в качестве данных Ini, который должен быть загружен после загрузки 'Файл по умолчанию (Default File)'. Это должно соблюдать все параметры объекта, за исключением сжатия или шифрования. То есть должно быть в несжатом и в не зашифрованном виде вне зависимости от настроек. Существует ограничение 3000 знаков.
    • Если Вы отметили пункт «Автозагрузка», то текущий файл загружается с диска каждый раз, когда он читается. Не рекомендуется использовать эту опцию, это медленно.
    • Если Вы отметили пункт «Автосохранение», то данные записываются в текущий файл каждый раз, когда он изменяется. Если в этом нет необходимости, то лучше не использовать эту опцию.
    • Если Вы отметили пункт «Создать папки для файлов, если они не существуют?» означает, что если вы пытаетесь сохранить файл Ini по пути который не существует, то папка будет создана автоматически. Это тоже работает рекурсивно. Если в этом нет необходимости, то не используйте эту опцию.
    • Если Вы отметили пункт «1 начальный индекс», то все выражения которые требуют номер (такие как 'Получить энное название группы (Get N-th Group name)') будут начинаться с 1. В противном случае они будут начинаться с нуля.
    • Если вы не хотите беспокоиться об остальных настройках, просто установите флажок 'Использовать стандартные настройки? (Use Standard Settings?)'. Тогда остальные настройки будут работать так как Вы ожидаете!

    Если Вы не отметили пункт «Использовать стандартные настройки?», то можно установить следующим образом:

    • Если Вы отметили пункт «Использовать сжатие?», файл будет загружаться и сохраняться с использованием сжатия.
    • Если Вы отметили пункт «Шифрование», файл будет загружаться и сохраняться с использованием шифрования. Ключ шифрования должен быть записан в поле.
    • Если Вы не отметили пункт 'Новая линия (New line)', то будет использоваться стандартные окна новой линии. В противном случае строка ввода в поле будет использоваться как новая линия строки.
    • 'Всегда указывать строку? (Always quote strings?) определяет, будут ли строки всегда указываются или нет в сохранённых файлах.
    • 'Чувствительно к регистру? (Case sensitive?) определяет, будет ли объект Ini++ чувствителен к регистру или нет. Это влияет загрузку и сохранение тоже.
    • Если Вы отметили пункт 'Разрешить пустые группы (Allow empty groups)', то объект может содержать пустые группы. В противном случае они просто удаляются.
    • Использование 'Загружать и сохранять подгруппы (Load and Save Subgroups)' описано в 'Подгруппы (Subgroups)' в разделе 'Расширенные возможности (Advanced Features)'.
    • Использование 'Повторяющиеся группы (Repeated Groups)', 'Повторяющиеся элементы (Repeated Items)' и 'Сохранить повторяющиеся элементы как оригинал? (Save repeated items as original?)' описано в 'Повторяющиеся Группы и Элементы (Group and Item Repeats)' в разделе 'Расширенные возможности (Advanced Features)'.
    • Использование 'Управляющие символы в названиях группы? (Escape characters in group names?)' описано в 'Управляющие символы (Escape Characters)' в разделе 'Расширенные возможности (Advanced Features)'.
