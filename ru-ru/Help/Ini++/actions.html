Основное меню (Main Menu)

Задать текущую группу

Изменяет текущую группу для использования с действиями и выражениями подменю 'Текущая группа(Current Group)'. Второй параметр позволяет Вам указать если объект должен обеспечить новое название указанной группе. Если он установлено на 0, то текущая группа установлена на первый параметр, вне зависимости от того какая.

Если второй параметр установлен на 1, то текущая группа устанавливается на первый параметр с '.x' добавляется к концу, где 'х' наименьшее число, которое означает, что группа не существует.

Например: Если первый параметр установлен на группу и второй параметр установлен на 1, то, если группы 'группа группа.1, группа.3' уже существуют, то текущая группа будет установлена в 'группу.2'.

К началу

Текущая группа(Current Group)

Все действия в этом подменю более идентичны основному варианту действий, за исключением параметра группа установить к текущей группе, как определено действием «Задать текущую группу».

Примечание: Действие 'Переименовать группу (Rename Group)' также изменит текущую группу на новое название группы.

К началу

Настройки элемента

Установить значение (Set Value)

Это действие устанавливает значение указанного элемента на число.

Третий параметр спрашивает указано ли число в четвёртом параметре и будет ли число целым или с плавающей запятой. Преимущество указания с плавающей запятой очевидно, потому что цифры после запятой будут сохранены. Недостаток выбора чисел с плавающей запятой является то, что если вы работаете с большим целым числом, оно может получится приблизительно равным. Если выбрано целое число, то будет больший диапазон значений, пока число не будет приблизительно равным.

Например: Если 1, 000, 002, 124 хранится в виде целого числа, оно может быть получено точно. Однако если был использован вариант с плавающей запятой, то фактически получили бы как 1, 000, 002, 112. Поэтому: Если вы просто хотите сохранить целые числа, используйте параметр целого числа.

Если используется режим целое число, большие целые числа могут 'обрезаны (wrap around)' до больших отрицательных чисел.

Когда режим с плавающей запятой используется, вы можете заметить, что некоторые числа сохраняются точно так, как и следовало ожидать. Это происходит потому, что некоторые числа не могут быть точно представлены в плавающей запятой.

Если число очень большое, оно может быть представлено в виде '1.#INF'.

Установить строку (Set String)

Это действие устанавливает значение указанного элемента в заданной строке.

Установить строку(MD5 хешировано) (Set String (MD5 Hashed))

Это действие устанавливает значение указанного элемента в хеш-значение MD5 из строки.

Что подразумевается под - фиксированная длина кода, который представляет собой строку. Например, некоторые строки хэш:

  • «Hello», хешированное 8b1a9953c4611296a827abf8c47804d7
  • «Ini++ is probably the best thing ever», хешированное a6da1d8f5556dce9c12bf9ba0de45000
  • «An ion is a charged atomic particle. The idea of the Trapped Ion system of Quantum Computing is to suspend an ion in free space in a vacuum. It is known that this cannot be done using static forces, however it can be done using oscillating forces. The oscillating electric fields form a saddle shape, which the ion will eventually get stuck in the middle of. The information is stored in the charge state of the ion. Lasers can be fired at the ions to change their states or cause interference.», хешированное 188c77b0329cb69619d15c77b87707e5

Очевидно что многие строки могут быть хешированы к одному значению, но все ещё остаётся большое количество различных возможных кодов. Может быть использовано для хранения пароля без чего либо, если пароль был изначально.

Хеширование чувствительно к регистру независимо от опций чувствительности к регистру.

Используйте условия элемента имеющего хешированное значение, чтобы увидеть, если значение конкретного элемента является хэшем из конкретной строки.

Сохранить объект (Save Object)

Разрешает некоторые атрибуты активного объекта (или объекта с аналогичным свойством к активному объекту), чтобы легко сохранить. Группа указана и элементы в этой группе хранят различные значения.

Второй параметр позволяет пользователю указать, если положение объекта сохраняется. Если этот параметр не равен нулю, то следующие элементы будут сохранены в группе:

  • posx: X позиция объекта
  • posy: Y позиция объекта
  • dir: Направление объекта
  • spd: Скорость объекта

Второй параметр позволяет пользователю указать, если переменные значения объекта сохранены. Если этот параметр не равен нулю, то следующие элементы будут сохранены в группе:

  • flags: Флаги объекта
  • 'valn': Энное переменное значение, 'n' между 0 и 25. Например 'val04'
  • 'strn': Энное переменное значение, 'n' между 0 и 25. Например 'str12'

Загрузить объект (Load Object)

Первый параметр позволяет указать объект. Второй параметр указывает группу. Различные элементы в этой группе будут загружены в объект. Значения этих элементов указано выше в разделе Сохранить объект (Save Object).

Если какие-либо значения отсутствуют, они не будут загружены.

Сохранить позицию (Save Position)

Сохраняет указанную позицию как строку в виде позиции X, позиции Y. Используйте выражения 'Получить позицию X (Get X Position)' и 'Получить позицию Y (Get Y Position)' что бы легко получить 'X' и 'Y' позиции.

Сохранить глобальные значения (Save Global Values)

Сохраняет глобальные значения и строки в указанную группу. Глобальные строки записываются в виде 'strn' и глобальных значений в виде 'valn'. Однозначные цифры значения 'n' будут иметь нулевой префикс, например 'str03' вместо 'str3'.

'vals' указывает число глобальных значений и 'strs' указывает число глобальных строк.

Загрузить глобальные значения (Load Global Values)

Загрузка глобальных значений и строк из указанной группы. Все представленные значения в виде 'strn' и 'valn' загружены (для 'n' по крайней мере две цифры в длину и меньше чем 'strs' и 'vals'). Число глобальных значений и строк будет динамически изменяться при необходимости.

Добавить повторяющийся элемент (Add Repeated Item)

'Добавить повторяющийся элемент (Add Repeated Item)' будет автоматически переименовывать название элемента (или название группы) на такое же, но уникальное название, так что данные могут быть легко сохранены без перезаписи чего либо.

Первый параметр указывает название группы, второй название элемента, третий имеется ли значение для элемента и будет определяться режим.

Если последний параметр равен 0, то действие сделает элемент уникальным, изменив название элемента. Он будет делать это путем добавления '.n' к концу названия, где 'n' является самым наименьшим целым числом, таким образом элемент не существует в указанной группе. Это не изменит название группы.

Если последний параметр равен 1, то действие сделает элемент уникальным, изменив название группы. Он будет делать это путем добавления '.n' к концу названия, где 'n' является самым наименьшим целым числом, таким образом группа уже не содержит элемент с указанным названием. Группа может существовать. Это не изменит название элемента.

Значение должно быть указано в виде строки. Используйте выражение 'Str' для преобразования числа в строку.

Смотрите также действие «Задать текущую группу», которое может установить текущую группу в новую группу, используя подобную систему переименования.

К началу

Перемещение и копирование (Moving and Copying)

Действия в этом меню позволяет перемещать и копировать элементы между группами и группами, которые будут переименованы и скопированы.

Переименовать группу (Rename Group)

Это действие переименует группу.

Третий параметр указывает что делать, если группа с новым названием уже существует. Это может быть одно из следующих действий:

0 Перезаписать (Overwrite) Существующая группа с названием назначения перезаписывается.

1 Соединить (Merge) Старая группа и новая группа соединяются вместе. Группа которая была переименована имеет приоритет, если обе группы имеют общие элементы.

2 Никогда не перезаписывать (Never Overwrite) Если группа уже существует с названием назначения, действие не выполняется.

Переименовать элемент (Rename Item)

Это действие переименует элемент внутри группы. Он не будет перемещён за пределы группы.

Четвёртый параметр определяет, разрешит ли эта операция перезаписывать другие части информации. Если он установлен, то перезапись не разрешается и уже есть элемент с названием назначения в группе, действие не выполняется.

Примечание: Переместить элемент (Move Item) это более общая версия этого действия.

Переместить элемент (Move Item)

Это действие позволяет перемещать элемент из одной группы в другую, и возможно изменит его название в этом процессе.

Пятый параметр определяет будет ли эта операция позволять другой части информации быть перезаписанной. Если установлен так что перезапись не разрешена и уже есть элемент с названием назначения в группе назначения, это действие не выполняется.

Примечание: Переименовать элемент (Rename Item) и Переместить элемент между группами (Move Item between groups) это менее общая версия этого действия.

Переместить элемент между группами (Move Item between groups)

Это действие позволяет перемещать элемент из одной группы в другую, сохраняя название элемента нетронутыми.

Четвёртый параметр определяет будет ли эта операция позволять другой части информации быть перезаписанной. Если установлен так что перезапись не разрешена и уже есть элемент с таким же названием в группе назначения, это действие не выполняется.

Примечание: 'Переместить элемент (Move Item)' это более общая версия этого действия.

Копировать группу (Copy Group)

Это действие позволяет скопировать завершенную группу в другую группу.

Третий параметр указывает, что эта операция будет делать в случае, если группа назначения уже существует. Предусмотрены следующие варианты:

0 Перезаписать (Overwrite) Существующая группа с названием назначения перезаписывается.

1 Соединить (Merge) Старая группа и новая группа соединяются вместе. Группа источник имеет приоритет, если обе группы имеют общие элементы.

Копировать элемент (Copy Item)

Это действие копирует элемент. Название назначения группы и название элемента могут быть указаны.

Это действие всегда будет перезаписывать любые существующие данные, нет никаких опций. Если вы хотите убедиться, что это действие не приводит к перезаписи данных, проверьте с помощью условия 'Группа имеет элемент (Group Has Item)'.

Сдвиг (Shift), Сортировка (Sort), Перемешивать/Микс. (Shuffle)

Как следует из названия, это действие может выполнять различные операции, такие как 'Сдвиг (Shift)' данных, 'Сортировка (Sort)' данных и 'Перемешивать/микс. (Shuffle)' данных.

Первый параметр выбирает операцию которая должна быть выполнена, а второй параметр выбирает группу для которой операция будет выполнена. Различные операции перечислены ниже. Следующая группа используется для примеров:

Примерный INI 11

[group]
Cat = Jess
Dog = Rex
Elephant = Elly

Сортировка/сорт. (Sort) Сортирует значения без изменения названия элементов. Например, пример будет отсортирован как:

Примерный INI 12

[group]
Cat = Elly
Dog = Jess
Elephant = Rex

Перемешивать/Микс. (Shuffle) Случайным образом меняет местами значения, но сохраняет названия элементов без изменений.

Сдвиг вверх/вниз (Shift Up/Down) 'Сдвиг вверх' сдвигает все значения вверх, отбросив первое значение и 'Удаление (Deleting)' последнего названия элемента. 'Сдвиг вниз' аналогичен.

Цикл верх/вниз (Cycle Up/Down) 'Цикл верх' сдвигает все значения вверх, и устанавливает последнее название элемента на первое старое значение (следовательно 'цикл' значения). 'Цикл вниз' аналогичен.

Реверс (Reverse) Меняет пары между названиями элементов и значениями. Например:

Example Ini 13

[group]
Cat = Elly
Dog = Rex
Elephant = Jess

Менять названия элементов и значений/Менять знач. и элем. (Swap items and values) Меняет названия элементов для их значений. Если есть повторы, используется последний. Например, на примере группы:

Example Ini 14

[group]
Elly = Cat
Jess = Elephant
Rex = Dog

Перенумеровать элементы/Пер-нумер.элем. (Renumber items) Изменение названия каждого элемента к числу. Например:

Example Ini 15

[group]
1 = Cat
2 = Elephant
3 = Dog

К началу

Удаление (Deleting)

Это меню содержит различные действия, которые могут удалять элементы.

Удалить группу (Delete Group)

Это действие удалит группу. Все элементы в группе будут удалены. Если группа не существует действие не выполняется.

Удалить элемент (Delete Item)

Это действие удалить элемент в группе. Если группа или элемент не существует действие не выполняется.

Если 'Разрешить пустые группы (Allow empty groups)' флажок не установлен (по умолчанию это не так), и элемент удален последним в группе, то вся группа будет удалена. В противном случае он будет по-прежнему существовать, но будет пустым.

Удалить элемент во всех группах (Delete Item in all groups)

Это действие удалит все элементы с указанным названием, независимо от того, в какой группе они находятся. Название элемента, который будет удалён первым и единственный параметр.

Удалить по шаблону (Delete by Pattern)

Это действие удалить все элементы, которые соответствуют поиску.

Шаблоны поиска могут быть заданы для названия группы, названия элемента и значения элемента. Это первый, второй и третий параметры, соответственно. Чтобы элемент был удалён, он должен соответствовать каждому шаблон поиска. В шаблоне поиска, '.' означает любой одиночный символ и '*' обозначает любую последовательность символов.

Последний параметр определяет, если поиск чувствителен к регистру. Даже если он установлен на 1, только названия групп и элементов будут чувствительны к регистру, если объект находится в режиме чувствительно к регистру.

Очистить всё (Clear All)

Удаляет все данные в объекте. Не изменяя текущий файл. Делая это в 'Новый файл (New File)'.

К началу

Поиск (Searching)

Это меню содержит различные действия, которые могут найти объект.

Выполнить поиск (Perform Search)

Это действие выполняет поиск. Он рассматривает каждый элемент в Ini++ (объект) (при условии оптимизации) и сравнивает название группы, название элемента и значение элемента для трех данных шаблонов. Если элемент совпадает со всеми из них (если не 'соответствует любому из этих условий' опция используется), он добавляется к результатам поиска. Результаты поиска могут быть доступны в выражениях в разделе меню выражений 'Результаты поиска (Search Results)'.

Шаблоны являются обычными строками. Тем не менее символ '.' (точка) обозначает любой символ, а символ '*' (звёздочка) обозначает любую последовательность символов. Например:

  • Шаблон 'Hello' совпадения 'Hello' и ничего больше.
  • Шаблон '.. Fred' совпадения ''Hi Fred', 'Go Fred', 'Yo Fred', и т. д..
  • Шаблон 'a*' совпадения 'apple', 'acorn' и любая строка, которая начинается с 'a'.
  • Шаблон '*a*' совпадения любой строки в котороя имеется 'a'.
  • Шаблон '*' любое совпадение.

Вы также можете поместить перед выражением восклицательный знак ('!') для того чтобы означало обратное. Например:

  • Шаблон '!a*' совпадения любой строки которая не начинается с 'a'.
  • Шаблон '!*a*' совпадения любой строки которая не содержит 'a'.
  • Шаблон '!*' ничего не совпадает.

Первый параметр это шаблон для группы. Второй шаблон название элемента, и третий шаблон для значения элемента.

Четвёртый параметр определяет, какие используются настройки. К ним относятся следующие:

Чувствительно к регистру (Case sensitive) Определяет будет ли поиск чувствителен к регистру. Когда объект Ini++ в режиме не чувствителен к регистру, эта опция будет применяться только к шаблону 'значение (value)', с двумя другими шаблонами не чувствительными к регистру. Когда Ini++ объект находится в режиме чувствителен к регистру применяется ко всем шаблонам.

Совпадение с любым из условий/Совпад. с условием (Match any of the conditions) Элемент добавляется к результатам поиска, если он совпадает с любым из трёх условий. Без этого, он должен совпадать со всеми. Примечание: Этот режим работает медленнее, чем другие режимы, так как он должен искать каждую группу.

Добавить название группы только к результатам/Добав.имя групп в результат (Add group name only to results) При выборе этой опции только название группы добавляется к результатам, если элемент находится в группе. Поля название элемента и значение остаются пустыми. Это полезно если цель поиска найти группы с определенным свойством. Также позволяет использовать 'and', 'or' и 'xor' в этих ситуациях.

Вы также можете выбрать как результаты соединяются со старыми результатами (если все). Возможности:

Удалить предыдущие результаты/Удал.пред. результат (Discard previous results) Старые результаты поиска удаляются до выполнения поиска.

Or Результаты соединяются со старыми результатами.

And Элемент будет только в результатах, если он был в обоих старых результатах и соответствует новому поиску.

Xor Элемент будет в результатах, если он был в старых результатах или если он соответствует новому поиску, но не в обоих случаях одновременно.

Когда элемент помещается в результатах поиска, весь элемент копируется в результатах поиска. Это означает что результаты могут быть получены, как именно они были получены когда поиск проводился, независимо от того если элементы были удалены или изменены. Используйте выражения в подразделе меню выражений 'Результаты поиска (Search Results)', чтобы получить эту информацию.

Результаты поиска могут быть удалены действием 'Очистить результаты (Clear Results)'. Действия 'Найти 'Повторяющиеся' элементы (Find 'Repeats' of Item)' и 'Найти 'Подгруппы' (Find 'Subgroups') могут также изменить результаты поиска. 'Найти' Повторяется 'пункта' и 'Найти' Подгруппы действия могут также изменить результаты поиска

Найти 'Повторяющиеся' элементы (Find 'Repeats' of Item)

Находит все элементы которые соответствуют повторениям элемента, когда режим повторения 'Переименовать (Rename)' включён. То есть он находит все в виде 'элемент(item). *' в указанной группе. Результаты поиска будут также включать в себя 'элемент(item)' если он существует.

Первый параметр название группы, второй параметр название элемента.

Поиск всегда чувствителен к регистру. Результаты помещаются в обычные результаты поиска, доступные из выражений. Предыдущие результаты всегда очищается перед выполнением действия.

Например, данный файл:

Example Ini 16

[group]
item = 4
item.3 = 21
item.hello = 343
item4 = 12
[group2]
item.4 = 42

Тогда результаты поиска когда это действие выполняется с группой 'группа' и элементом 'элемент' будет выглядеть так:

Example Ini 17

[group]
item = 4
item.3 = 21
item.hello = 343

Найти 'Подгруппы'

Находит названия всех групп, которые соответствуют подгруппе другой группы, когда включена опция 'Разрешить подгруппы (Allow Subgroups)'.

Первый параметр название группы, чтобы найти подгруппы. Это может группа по подгруппе (т.е. один с '/' в его названии).

Вторым параметр количество уровней. Если это положительное число, он находит все подгруппы на этом уровне или ниже. Если это отрицательное число, он находит все подгруппы точно на таком же уровне. Если он равен 0, если находит все подгруппы на любой глубине.

Например, рассмотрим следующий Ini++ файл с включёнными подгруппами:

Example Ini 18

[parent]
[a]
item = 1
[aa]
item = 2
[ab]
item = 2
[aba]
item = 3
[abb]
item = 3
[abba]
item = 4
[b]
item = 1
[bb]
item = 2

(Элемент называется элементом в каждой группе в действительности глубина этой группы по отношению к родителю)

Тогда, если вы:

  • Найти подгруппы 'родителя (parent)' с глубиной 1, Вы получите: a, b
  • Найти подгруппы 'родителя (parent)' с глубиной 2, Вы получите: a, aa, ab, b, bb
  • Найти подгруппы 'родителя (parent)' с глубиной -2, Вы получите: aa, ab, bb
  • Найти подгруппы 'родителя (parent)' с глубиной -4, Вы получите: abba
  • Найти подгруппы 'a' с глубиной 2 Вы получите: aa, ab, aba, abb
  • Найти подгруппы 'родителя (parent)' с глубиной 0, Вы получите: a, aa, ab, aba, abb, abba, b, bb

Для этого примера, приведены полные названия групп которые не были использованы, чтобы сделать его более легким. Например: aba в действительности называется группа/a/ab/aba. Это последний, который придёт в результатах поиска.

Результаты помещаются в обычных результатах поиска, доступными выражениями. Предыдущие результаты поиска будут удалены в первую очередь. Названия элементов и значения элементов для всех результатов поиска будут все пусты, так как они не имеют отношения.

Очистить результаты (Clear Results)

Очищает текущие результаты поиска. Нет необходимости для этого (как и любое поисковое действия либо автоматически, либо даёт возможность очистить текущие результаты), но это могло бы помочь с логикой вашей программы.

Найти и поместить результат в Ini файл (Search and put result in Ini file)

Это как действие 'Выполнить поиск (Perform Search)', однако результат сохраняется в виде файла Ini по указанному пути. Этот файл Ini будет содержать только те элементы, которые выполняют поиск. Настройки для файла Ini будут взяты из объекта.

Первый параметр название файла для сохранения. Это может быть относительный путь к файлу. В таком случае, это будет относительно каталога по умолчанию.

Остальные параметры, как и с действием 'Выполнить поиск (Perform Search)'.

Это действие не влияет на результаты поиска.

Найти и поместить результат в Ini++ объект (Search and put result in Ini++ object)

Это как действие 'Выполнить поиск (Perform Search)', однако все совпавшие элементы копируются в другой Ini++ объект.

Первый параметр название Ini++ объекта для копирования данных в. Это название объекта, как установлено в редакторе сцены. Он должен быть введён точно, чувствительно к регистру!

Целевой Ini++ объект не может быть таким же как Ini++ который генерирует действие. Кроме того, он не может обмениваться данными с ним. Это означает, что они оба не могут быть глобальными объектами. Когда программа запущена из редактора событий если любое из описанных условий происходит, или указанный Ini++ объект не существует на экране появится окно с сообщением. Когда программа будет создана, то не будет никакого сообщения об ошибке и действия тоже не будет.

Остальные параметры, как и с действием 'Выполнить поиск (Perform Search)'.

Это действие не влияет на результаты поиска.

Соединить (Merging)

Это меню содержит различные действия, которые позволяют одному Ini++ объекту соединиться с другим.

Соединить (с файлом) (Merge (with file))

Это действие соединяет файл и текущие данные в Ini++ объекте.

Первый параметр файл для соединения с. Это может быть относительный путь, и это будет относительно пути по умолчанию.

Второй параметр определяет, разрешена ли перезапись. Если параметр установлен на 1, то все данные копируются из файла в объект. Если параметр установлен на 0, то данные копируются из файла в объект если только это не переписывает существующие данные.

Генерирует условие события 'При соеденении (On Merge)'.

Соединить группу (с файлом) (Merge Group (with file))

Соеденяет указанные группы из файла с текущими данными.

Первый параметр файл для соединения с. Это может быть относительный путь, и это будет относительно пути по умолчанию.

Второй параметр запрашивает название группы в файле Ini.

Третий параметр запрашивает название назначения группы. Это то как группа будет называться в Ini++ объекте. Часто вы просто хотите, чтобы это было так же, как второй параметр.

Четвёртый параметр определяет разрешена ли перезапись. Если не разрешена (то есть, этот параметр имеет значение 0), то данные копируются из файла в объект, если только это не переписывает существующие данные.

Генерирует условие события 'При соеденении (On Merge)'.

Соединить (с другим Ini++ объектом) (Merge (with other Ini++ object))

Это действие соединяет два Ini++ объекты вместе, и помещает результат в вызываемый объект.

Первый параметр название Ini++ объекта для соединения с. Это название объекта как установлено в редакторе сцены. Он должен быть введён точно, чувствительно к регистру. Указанный Ini++ объект не может быть таким же как Ini++ который генерирует действие. Кроме того, он не может обмениваться с ним данными. Это означает, что они оба не могут быть глобальными объектами. Когда программа запущена из редактора событий если любое из описанных условий происходит, или указанный Ini++ объект не существует на экране появится окно с сообщением. Когда программа будет создана, то не будет никакого сообщения об ошибке и действия тоже не будет.

Объект, который будет введён для этого параметра не будет изменён. Только вызываемый объект будет изменён.

Второй параметр определяет, будут ли данные из указанного объекта перезаписать данные в вызываемом объекте. Если установлено значение 0, то в действительности вызываемый объект получает приоритет, если есть элементы с тем же названием и в той же группе в обоих объектах. В противном случае, если он установлен на 1, то объект, указанный в первом параметре выполняет действие.

Генерирует условие события 'При соеденении (On Merge)'.

Соединить группу (с другим Ini++ объектом) (Merge Group (with other Ini++ object))

Это действие соединяет вместе указанную группу в двух Ini++ объектах, и помещает результат в вызываемый объект.

Первый параметр название Ini++ объекта для соединения с. Это название объекта как установлено в редакторе сцены. Он должен быть введён точно, чувствительно к регистру. Указанный Ini++ объект не может быть таким же как Ini++ который генерирует действие. Кроме того, он не может обмениваться с ним данными. Это означает, что они оба не могут быть глобальными объектами. Когда программа запущена из редактора событий если любое из описанных условий происходит, или указанный Ini++ объект не существует на экране появится окно с сообщением. Когда программа будет создана, то не будет никакого сообщения об ошибке и действия тоже не будет.

Объект, который будет введён для этого параметра не будет изменён. Только вызываемый объект будет изменён.

Второй параметр запрашивает название группы в указанном объекте для соединения.

Третий параметр запрашивает название назначения группы. Это то как группа будет называться в Ini++ объекте. Часто вы просто хотите, чтобы это было так же, как второй параметр.

Последний параметр определяет, будут ли данные из указанного объекта перезаписывать данные в вызываемом объекте. Если установлено значение 0, то в действительности вызываемый объект получает приоритет, если есть элементы с тем же названием в обоих объектах. В противном случае, если он установлен на 1, то объект, указанный в первом параметре выполняет действие.

Генерирует условие события 'При соеденении (On Merge)'.

К началу

Загрузка и сохранение (Loading and Saving)

Это меню содержит различные действия, для загрузки и сохранения.

Изменить текущий файл (Change Current File)

Это действие изменяет название текущего файла, возможно удаление всех данных.

Первый параметр является названием нового файла. Это может быть относительный путь. Если это оставить пустым (т.е. это пустая строка) то файл хранится как внутренний.

Второй параметр определяет должны ли текущие данные быть очищены или нет. Установка на 0 сохраняет текущие данные. Установка на 1 очищает текущие данные. Если выбрано 0 и автосохранение включено, то файл будет немедленно сохранён. Если выбрано 1 то файл не будет сохранён независимо от настроек автосохранения.

Статус только для чтения не зависит от этого действия.

Это действие отличается от 'Загрузить файл (Load File)' поскольку он фактически не загружает файл. Он просто устанавливает его в качестве текущего файла для автосохранения и автозагрузки.

Загрузить файл (Load File)

Это действие удаляет все текущие данные и загружает как INI файл.

Файл загружается на основе текущих настроек. Включая настройки сжатия и шифрования. Убедитесь, что вы загрузите файл с правильными настройками!

Второй параметр указывает, следует ли открывать файл в режиме только для чтения. Это означает что обе функции «Сохранить» и функция автосохранения выключены. Данные могут быть изменены в памяти.

Если выбрана опция 'Создать папки для файлов, если они не существуют? (Create folders for files if they do not exist?)' то папки будут созданы для файла, он может быть создан если не выбран режим только для чтения. Если было выбрано только для чтения, папки не будут созданы независимо от настроек.

Сохранить файл (Save File)

Это действие сохраняет Ini++ объект в текущий файл. Все настройки соблюдаются.

Действие потерпит неудачу, если файл находится в режиме только для чтения, или нет никакого текущего файла (который в случае, если «Новый» было использовано с пустой строкой в качестве параметра, или если 'Новый (New)' или 'Загрузить (Load)' не были использованы и файл не был указан в редакторе сцены).

Сохранить как (Save As)

Это действие сохраняет Ini++ объект в указанный файл. Все настройки соблюдаются.

Текущий файл будет установлен на указанное название файла. Это может быть относительный путь.

Действие будет работать, даже если файл находится в режиме только для чтения. Если файл был успешно сохранён, режим «только чтение» будет выключен.

Резервная копия в (Backup To)

Это действие сохраняет объект Ini++ в файл. Настройки сжатия и шифрования могут быть заданы.

Первым параметр название файла. Это может быть относительный путь, и он будет относителен каталога по умолчанию.

Второй параметр определяет должен ли файл быть сжат.

Третий параметр является ключом шифрования. Если он пустой (то есть пустая строка), то шифрование не используется.

Это действие не изменяет текущий путь к файлу. Действие будет работать, даже если файл находится в режиме только для чтения. Это не изменит режим только для чтения.

Закрыть (Close)

Очищает все данные и устанавливает текущий путь к пустой строке, что делает файл внутренним.

Импортировать/экспортировать CSV (Import/Export CSV)

Позволяет импортировать или экспортированы данные в CSV виде. CSV означает 'значения разделённые запятыми (comma-separated values)' это просто список значений разделенных запятыми (или другим символом по вашему выбору).

Первый параметр спрашивает, хотите ли импортировать или экспортировать. Введите 0, если вам нужно импортирование, и 1, если экспортирование.

Это действие не влияет на текущий путь к файлу.

Импортирование (Importing)

Импортирует данные из файла CSV.

Второй параметр название файла для загрузки из. Это может быть относительный путь.

Третий параметр название группы для загрузки. Если оставить пустым, то каждая группа будет находиться на своей собственной строке и первое значение в каждой строке будет название группы.

Четвёртый параметр является разделителем. Как правило это запятая, но если это может привести к возникновению запятых в ваших данных, то вы можете изменить его на символ который вы не используете. Например, в 'pipeline' символ 'k' не используется и может стать разделителем.

В качестве примера, рассмотрим следующий CSV-файл (с разделителем «,»):

Group 1|Item 1|Value 1|Item 2|Value 2
Group 2|Item 3|Value 3|Item 4|Value 4|Item 5|Value 5

Если настройки группы оставить пустыми, она будет загружена как:

Example Ini 19

[Group 1]
Item 1 = Value 1
Item 2 = Value 2
[Group 2]
Item 3 = Value 3
Item 4 = Value 4
Item 5 = Value 5

Если параметр группы устанавливается как 'Группа (Group)', то он будет загружен следующим образом:

Example Ini 20

[Group 1]
Group 1 = Item 1
Value 1 = Item 2
Group 2 = Item 3
Value 3 = Item 4
Value 4 = Item 5

(Примечание: Если есть элемент без данных в конце строки, он игнорируется) Это действие не удаляет старые данные в первую очередь.

Экспортирование (Exporting)

Экспортирует данные как файл CSV.

Вторым параметром является название файла для сохранения. Это может быть относительный путь.

Третий параметр является названием группы для сохранения. Если оставить пустым, то каждая группа будет находиться на своей собственной строке и первое значение в каждой строке будет название группы. В противном случае только указанная группа будет сохранена и её название не будет приставкой к строке.

Четвёртый параметр является разделителем. Как правило это запятая, но если это может привести к возникновению запятых в ваших данных, то вы можете изменить его на символ который вы не используете. Например, в 'pipeline' символ 'k' не используется и может стать разделителем.

Настройки, такие как 'Сохранить повторяющиеся элементы как оригинал (Save repeated items as original)' не соблюдаются и управляющие символы не используются.

Используйте выражение 'Получить INI как CSV строку (Get INI as CSV String)', чтобы получить эти данные без сохранения его на диск.

Импортировать/экспортировать массив CSV (Import/Export CSV Array)

Это действие может импортировать или экспортировать данные в виде массива CSV. Это CSV файл в котором верхняя строка определяет возможные элементы для каждой группы, и следующие строки представляют каждую отдельную группу.

Первый параметр определяет должен ли файл быть импортированы или экспортированы в. Второй параметр определяет файл (который может быть относительным) и последний параметр определяет разделитель (обычно запятая).

Например, предположим что у нас есть данные:

Example Ini 21

[Group 1]
item1 = Hat
item2 = Toothbrush
item3 = Lipstick
item4 = Key
[Group 2]
item1 = Glass
item2 = Chewing Gum
item3 = Hammer

Тогда данные будут экспортированы как:

Group,item1,item2,item3,item4
Group 1,Hat,Toothbrush,Lipstick,Key
Group 2,Glass,Chewing Gum,Hammer

Обратите внимание на то, как первая строка определяет названия элементов. Каждая строка имеет следующие элементы, с первым элементом это название группы. Примечание: первая строка определяет названия элементов. Каждая строка имеет эти элементы, с первым элементом их названием группы.

Импортирование похоже. Данные не очищается перед импортированием. Если элемент пуст, он не будет загружен. Например:

Group,item1,item2,item3,item4
Group 1,Hat,Toothbrush,,Key

Будет загружен как:

Example Ini 22

[Group 1]
item1 = Hat
item2 = Toothbrush
item4 = Key

Так как третий элемент пустой. Он не будет загружен как пустая строка

Режимы не изменяют текущий путь к файлу.

Загрузить из строки (Load From String)

Это действие загружает INI файл из строки. Строка должна быть отформатирована как файл INI. Все настройки (включая выбор символ новой строки) соблюдаются, когда это загружается кроме сжатия и шифрования.

Первый параметр строка, которую вы хотите загрузить. Второй параметр определяет, если вы хотите очистить все старые данные (0), или если вы хотите соединить новые данные со старыми данными (1). В случае соединения новые данные заменят любые старые данные, если они имеют одни и те же группы и название элементов.

Изменить слот глобальных данных (Change Global Data Slot)

Это действие изменяет текущий выбранный глобальный слот данных. Если слот не существует, он будет создан.

Если объект не был создан в режиме 'Глобальные (global)', то он будет преобразован в глобальный режим. Это означает, что локальные данные будут удалены из памяти. Поэтому убедитесь что вы сохранили его, если вы хотите сохранить его!

Установить авто сохранение/загрузку (Set Auto Load/Save)

Это действие позволяет включать и выключать автоматическую загрузку и автоматическое сохранение. Это не изменяет данные в любом виде.

Установить сжатие (Set Compression)

Это действие позволяет включать и выключать сжатие (для автозагрузки, автосохранения, «Загрузка (Load)», «Сохранение (Save)» и «Сохранить как (Save As…)»). Это не изменяет данные, но может изменить метод сохранения файла. Поэтому файл будет повторно сохранён.

Установить шифрование (Set Encryption)

Это действие позволяет установить ключ шифрования. Если вы оставите ключ пустым, шифрование будет выключено. Ключ шифрования используется для автозагрузки, автосохранения, «Загрузка (Load)», «Сохранение (Save)» и «Сохранить как (Save As…)». Это не изменяет данные, но может изменить метод сохранения файла. Поэтому файл будет повторно сохранён.

Установить только чтение (Set Read Only)

Это действие позволяет изменить статус объекта на только чтение. Это не изменяет данные или путь сохранения данных. Независимо как установлен этот параметр объект может быть модифицировано в памяти. Режим «только для чтения» выключает действия автосохранения и сохранения.

Диалоговое окно (Dialog Box)

Ini++ объект имеет встроенное диалоговое окно, которое может помочь при отладке приложений. Окно может также служить в качестве экрана читов, продвинутого редактора настроек или редактора данных в выпущенной программе.

Открыть диалог (Open Dialog)

Это действие открывает диалоговое окно.

Одновременно только один диалог можно запустить. Если другое диалоговое окно открыто, оно будет закрыто.

Первый параметр позволяет установить текст строки заголовка для диалогового окна. Второй параметр позволяет установить различные настройки. Следующие настройки:

  • Запретить удаление групп/Запрет удаление групп (Disallow deleting groups)
    Группы не могут быть непосредственно удалены. Примечание: Если 'Запретить удаление элементов/Запр. удал. элементов (Disallow deleting items)' не выбран, и 'Разрешить пустые группы (Allow empty groups)' опция не включена, то группа может быть удалена путём удаления всех элементов в ней.
  • Запретить удаление элементов/Запр. удал. элементов (Disallow deleting items)
    Элементы не могут быть непосредственно удалены. Примечание: Если 'Запретить удаление групп/Запрет удаление групп (Disallow deleting groups)' не выбран, то элемент может быть удалён путём удаления группы чьей частью он является.
  • Запретить переименование групп/Запр. переимен. групп (Disallow renaming groups)
    Группы не могут быть переименованы.
  • Запретить переименование элементов/Запр. переимен.элемент (Disallow renaming items)
    Элементы не могут быть переименованы.
  • Значения только чтение (Values are read-only)
    Значения элементов не могут быть изменены.
  • Не 'Обновлять' опция меню/Не 'Обновлять' опция (No 'Refresh' menu option)
    Меню правой кнопкой мыши не будет включать в себя опцию 'Обновлять(Refresh)', которая позволит пользователю обновить объект.
  • Запретить добавление групп/Запрет добав. групп (Disallow adding groups)
    Новые группы не могут быть добавлены.
  • Запретить добавление элементов/Запрет добав.элементов (Disallow adding items)
    Элементы не могут быть добавлены к группам. Примечание: Если не выбран 'Запретить добавление групп/Запрет добав. групп (Disallow adding groups)', то один пункт по-прежнему будет добавлен к любой новой группе.
  • Не обновлять значения элемента/Не обновлять знач. элемента (Do not refresh item values)
    Когда этот флажок не установлен, то объект будет автоматически обновлять все значения при событии 'При изменении данных (On Data Modified)'. Это не относится к новыми, удаленным или переименованным элементам. Если объект будет постоянно изменяться, то это может значительно замедлить программу.
  • Пауза программы пока открыты (Pause program whilst open)
    Если этот флажок установлен, программа будет приостановлена, пока диалоговое окно будет закрыто. В противном случае он будет работать в новом потоке, в фоновом режиме. Если этот флажок установлен, опции связанные с обновлением не имеют никакого смысла, и поэтому не могут быть выбраны.

В выпадающем меню в нижней части диалогового окна позволяет форматировать элементы в виде дереве для выбора.

Диалоговое окно автоматически выключается при смене сцены, когда связанный с ним объект перестаёт существовать.

Закрыть диалог (Close Dialog)

Если диалоговое окно открыто, то это действие закрывает его. Это вызовет условие события 'При закрытии диалогового окна(On Dialog Closed)'.

Если диалоговое окно не открыто, это действие не выполняется.

Обновить диалог (Refresh Dialog)

Если диалоговое окно открыто, выполняется его полное обновление. Это может отвлекать пользователя, а при больших объемах данных обновление может быть медленным.
