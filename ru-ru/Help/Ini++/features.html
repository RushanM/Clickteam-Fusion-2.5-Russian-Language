Сжатие (Compression)

Сжатие Хаффмана можно использовать для сжатия файла Ini. Лучше всего это работает для больших файлов. Обратите внимание, что возможно сжатый файл может быть больше несжатого файла. Это верно для любого типа сжатия.

Ini++ не может автоматически определить файл Ini сжат или нет. Опция 'Использовать сжатие? (Use Compression?)' должна быть включена при загрузке сжатого файла, чтобы он был загружен правильно. Загрузка несжатого файла с включённой опцией 'Использовать сжатие? (Use Compression?)' не позволит файлу работать правильно.

Если Ini файл загружается с выключённой опцией 'Использовать сжатие? (Use Compression?)', опция всё ещё может быть позже включена, так что последующие сохранения сжимаются. То же самое применимо с точностью до наоборот. Действие 'Резервная копия в (Backup to)' позволяет сохранить Ini как сжатый, не меняя всех настроек.

Обратите внимание, что данные не будут сжаты в памяти.

Настройки сжатия не влияют на действия или отладчик 'Загрузить из строки (Load From String)'.

Шифрование (Encryption)

INI файлы могут быть зашифрованы. Ключ указывается, используемый для шифрования ключ необходимо использовать и для расшифровки. Если указан неверный ключ файл будет неправильно расшифрован.

Шифрование не легко взломать, но не рекомендуется сохранять конфиденциальные данные в этом режиме. Метод шифрования никогда не изменит размер файла. Шифрование выполняется перед сжатием (если включены оба), и таким образом шифрования может сделать сжатие менее эффективным.

Ini++ не может автоматически определить файл Ini зашифрована или нет. Опция 'Шифрование (Encryption)' всегда должна быть включена и указан правильный ключ при загрузке зашифрованного файла, чтобы файл был загружен правильно. Загрузка не зашифрованного файла с включённой опцией 'Шифрование (Encryption)' не позволит файлу работать правильно.

Если Ini файл загружается без шифрования, опция всё ещё может быть включена позже, так что последующее сохранения сжимаются. То же самое применимо с точностью до наоборот, а ключ может быть изменён. Действие 'Резервная копия в (Backup to)' позволяет сохранить Ini в зашифрованном виде, не меняя всех настроек.

Обратите внимание, что данные не шифруются в памяти. Данные могут быть извлечены с помощью программы, которая может считывать память компьютеров напрямую. Ключ также может быть извлечён по такому же принципу. Также ключ может быть сохранён в виде простого текста, если он был закодирован в Multimedia Fusion.

Настройки шифрования не влияют на действия или отладчик 'Загрузить из строки (Load From String)'.

Повторяющиеся Группы и Элементы (Group and Item Repeats)

В Ini файле возможно появление группы дважды. Также это возможно для элемента, появляться дважды в той же самой группе. Например,

Example Ini 4: The file before it is loaded before Ini++

[group]
1=one
1=single
2=two
[group]
3=three

Как правило, в этом случае первый экземпляр только для чтения. Таким образом, только первый экземпляр группы будет читаться и внутри групп будут использоваться только первый экземпляр каждого значения. Ini++ имеет дополнительные настройки для изменения этого параметра.

Группы (Groups)

Есть четыре варианта что делать, если есть несколько групп с одинаковым названием:

First (Первая) Принимает только первый экземпляр, как это делает стандартный объект Ini

Last (Последняя) Принимает только последний экземпляр

Merge (Соединить) Соединяет группы вместе. См. ниже.

Rename (Переименовать) Переименует дополнительные группы, чтобы они имели уникальные значения. См. ниже.

Рассмотрим следующий пример файла, который имеет повторяющеюся группу (но для простоты, не повторяющиеся элементы):

Example Ini 5: The file before it is loaded before Ini++

[group]
1=a
2=b
3=c
[group]
4=d

При установке 'Merge (Соединить)', группы обрабатываются как одна. Таким образом, пример будет загружен как:

Example Ini 6: Example 5 after it is loaded before Ini++ using 'Merge' group repeat mode

[group]
1=a
2=b
3=c
4=d

Это вероятно самый полезный режим, и по умолчанию несмотря на его поведение будет отличается от стандартного Ini объекта.

Example Ini 7: Example 5 after it is loaded before Ini++ using 'Merge' group repeat mode

[group]
1=a
2=b
3=c
[group.1]
4=d

Действие 'Выполнить поиск (Perform Search)' можно использовать для поиска элементов в подобных группах.

Элементы (Items)

Повторяющиеся элементы работает аналогичным образом, хотя 'Merge (Соединить)' не имеет смысла в этом контексте.

First (Первая) Принимает только первый экземпляр, как это делает стандартный объект Ini

Last (Последняя) Принимает только последний экземпляр

Rename (Переименовать) Переименует последующие элементы в виде 'Название.n (Name.n)', где 'n' представляет собой число. Действие 'Найти 'Повторяющиеся' элементы (Find 'Repeats' of Item)' может легко искать всё элементы в том же виде.

Если используется опция 'Сохранить повторяющиеся элементы как оригинал? (Save repeated items as original)', то любой элемент в виде 'item.anything' будет сохранён так же как 'элемент (item)'. Это не влияет на данные в памяти.

Указывать строки (Quoting Strings)

Значения в Ini файлах могут заключить в двойные кавычки ('''). Это важно, если элемент имеет начальные или завершающие пробелы, так как они игнорируются. Стандартный Ini объект правильно читает строки в кавычках, но никогда не записывает их. Это приводит к потере начальных и конечных пробелов при сохранении данных с использованием стандартного Ini объекта.

Ini++ всегда будет указывать строки, когда требуется. С выбранным параметром 'Всегда указывать строки? (Always Quote Strings?)', строки будут указываться даже если этого не требуется. В противном случае, строки не будут указываться, если это не требуется.

С помощью объекта Стандартный Ini, названия групп обрезаны так, чтобы они никогда не начинались или заканчивались пробелом. В Ини ++, этого не происходит, независимо от режима, и никакие кавычки не нужны.

Управляющие символы (Escape Characters)

Предположим, что вы хотите иметь элемент под названием '1=1'. Тогда ваш Ini файл будет выглядеть так:

[group]
1=1 = value

Тем не менее, это было бы прочитано неправильно как представляющие элемент с названием '1' и значением '1 = значение'. Обычно нет никакого способа обойти эту проблему, но Ini++ добавляет управляющие символы.

Если управляющие символы включены, '\=' представляет собой знак равенства, но без этого значения. Поэтому когда управляющие символов включены в названии элемента, мы можем представить его правильно как:

[group]
1\=1 = value

Ниже приведен список поддерживаемых управляющих символов:

\n Новая строка¹

\t Символ табуляции (Tab), который в противном случае удаляется в виде пустого пространства в начале или в конце названия элемента

\_ Символ пробела (Space), который в противном случае удаляется в виде пустого пространства в начале или в конце названия элемента

\q Символ кавычек, поэтому значения могут начинаться и заканчиваться с символом кавычки. Также работает \'.

\= Действует только в названиях элементов

\; Действует только в начале названий элементов, чтобы строка не стала комментарием

\[ Действует только в начале названий элементов, чтобы предотвратить это интерпретируется как группа

Пожалуйста, обратите внимание, что управляющие символы обрабатываются автоматически. При использовании выражения 'Получить (Get)', управляющие символы автоматически поворачиваются назад к тому, что они представляют. При использовании действий 'Установить (Set)', Ваша строка автоматически получает управляющие символы, когда это необходимо.
Управляющие символы для групп, элементов и значений могут быть установлены по отдельности.

¹ Это представляет собой новую линию в качестве новой строки символов установленных в настройках, не обязательно разрыв строки.

Хеширование (Hashing)

'Хеширование (Hashing)' превращает строку в число не реверсивным способом. Это хорошо для хранения паролей. Например,

1. Пользователь выбирает пароль для своего входа в систему. Он хранится с помощью действия 'Установить строку (MD5 хешировано) (Set String (MD5 Hashed))'. Это означает, что пароль не сохраняется, только число представляющее его, а пароль не может быть выработан из этого.

2. На другой день пользователь хочет войти в систему. Он снова пишет свой пароль и пароль сравнивается со старым, используя условие 'Элемент имеет хешированное значение (Item Has Hashed Value)'.

3. Если они совпадают ему разрешено войти. В противном случае, неверный пароль и вход не разрешён.

Если это будет сделано, то файл Ini должен быть зашифрован. В противном случае, желающий получить доступ к учетной записи может открыть Ini файл и изменить запись на собственное значение, там где строка хеширована к этому значению. Таким образом он может изменить пароль. Шифрование означает что он не может просто изменить файл на то что он хочет.

Подгруппы (Subgroups)

Иногда желательно для группы Ini иметь подгруппы. Это не является стандартной функцией Ini файлов, но Ini++ позволяет легко это смоделировать.

В файле, в удобном виде для подгруппы обозначается вкладками. Например, Вы могли бы представить себе файл следующим образом. Здесь, 'родитель' имеет элемент, называемый 'элемент' и имеет подгруппу 'а'. В 'a' есть элемент называемый 'элемент' и две подгруппы, которые называется 'aa' и 'ab'. Это показывает, как подгруппы могут иметь свои подгруппы.

Example Ini 8: An example of subgroups

[parent]
item = 0
[a]
item = 1
[aa]
item = 2
[ab]
item = 2
[aba]
item = 3
[abb]
item = 3
[abba]
item = 4

Теперь, если Вы запросили Ini++ загрузить этот файл, он будет выглядеть как показано ниже в примере 9. Это происходит потому, что вкладки в Ini файлах обычно игнорируются как пустое пространство. Тем не менее, это специальная опция, загрузки и сохранения подгрупп, которая будет загружать этот файл в более полезном виде.

Example Ini 9: Example 8 when loaded without the Load and Save Subgroups option

[parent]
item = 0
[a]
item = 1
[aa]
item = 2
[ab]
item = 2
[aba]
item = 3
[abb]
item = 3
[abba]
item = 4

С опциями загрузки и сохранения подгрупп, каждая подгруппа будет переименована в путь, представляющий её. Пример может быть загружен как показано в примере 10 ниже:

Example Ini 10: Example 8 when loaded with the Load and Save Subgroups option

[parent]
item=0
[parent/a]
item=1
[parent/a/aa]
item=2
[parent/a/ab]
item=2
[parent/a/ab/aba]
item=3
[parent/a/ab/abb]
item=3
[parent/a/ab/abb/abba]
item=4

Действия такие как 'Найти подгруппы (Find Subgroups)', которое выведет список всех подгрупп группы. Это действие имеет опции, так что Вы можете найти прямые подгруппы, все подгруппы, первые 3 уровня подгрупп, которые являются именно 'правнуком (great grandson)' групп группы, или любой другой. Имейте в виду эту методику!
