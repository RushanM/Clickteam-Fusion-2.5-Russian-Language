Введение

Ini-файлы простые текстовые файлы, которые используются для хранения данных программ. Традиционно Ini-файлы используются для хранения настроек программы (Ini короткий для инициализации), однако сейчас для этого часто используется реестр. Это формат, который легко редактировать и с которым просто работать, часто является хорошим выбором для хранения данных в Multimedia Fusion.

Multimedia Fusion всегда имел объект для чтения и записи Ini файлов. Ini++ это просто его обновление, включающее множество новых возможностей.

Ini файлы

Файл Ini состоит из нескольких групп. В каждой группе несколько элементов. Начало группы обозначается названием группы в квадратных скобках. Элементы состоят из их названий, за которым следует знак равенства ('='), а затем соответствующее значение, связанное с ним. Например,

Example Ini 1: An Example File

[goals]
Henry=21
Hleb=4
Fabragas=2
[yellow_cards]
Rosisky=2
Hleb=3
Fabragas=5

Здесь есть две группы, голы(goals) и жёлтые карточки(yellow cards). Группы не должны иметь одинаковые названия элементов в каждой из них. В группе 'goal' есть три элемента: Henry, Hleb и Fabragas. Они имеют соответствующие значения 21, 4 и 2 соответственно. Аналогично для группы 'yellow cards'.

INI файлы в принципе такие же простые. Однако, Ini++ имеет опции для добавления дополнительных функций к ним, которые могут сделать их немного более сложным.

К началу

Что нового?

Ini++ имеет множество функций которые стандартный объект Ini не имел. Они перечислены ниже:

Сжатие и шифрование (Compression and Encryption) Ini++ позволяет сжимать и шифровать файлы при сохранении на диск.

Функции Пути (Path Features) Вы можете выбрать путь который относителен путям, относительно к. Вы можете сохранять в каталогах, которые не существуют, и будут созданы. 'Режим только чтение (read-only mode)', если вы не хотите изменять файл.

Начальные данные (Initial Data) Можно указать исходные данные, которые будут загружены в объект в редакторе сцены, а также исходный файл для загрузки.

Dialog Box Ini++ имеет диалоговое окно, которое можно открыть во время выполнения. Это полезно для отладки или чит меню.

Чувствительно к регистру (Case sensitive) Если вы хотите, файл Ini может быть чувствителен к регистру.

Повторяющиеся группы и элементы (Repeated Groups and Items) Повторяющиеся группы могут быть объединены вместе. Повторяющиеся группы и элементы также могут быть автоматически переименованы.

Инструменты отладки (Debugging Tools) Просмотр и изменение файла Ini как текста из Multimedia Fusion Debugger(отладчик).

Автосохранение (Auto-Save) Ini++ может быть настроен таким образом, чтобы автоматически не перезаписывает файл на диск, когда он был изменён, однако при необходимости это можно изменить.

Без ограничений! Нет никаких ограничений на размер файлов Ini, количества групп, количества элементов, размер значений или что-нибудь подобное!

Поиск (Searching) Есть действия и выражения для поиска в Ini файлах и даже помещения результатов в другие объекты Ini++.

Функция соединения (Merging Functions) Есть новые функции для объединения вместе Ini файлов.

Подгруппы (Subgroups) Группы с подгруппами могут быть смоделированы с объектом.

Хранилища глобальных данных Бесконечное количество хранилищ глобальных данных, которые могут быть общими для всех под-приложений.

Хешированное значение (Hashed Values) Есть функции, для значений имеющих хэшированные данные, которые полезны для паролей и т. д.

Получить и установить как текст Ini файлы могут быть загружены непосредственно в виде текста, или данные могут быть получены в виде строки.

Список групп и элементов (List Groups and Items) Выражение для перечисления элементов Ini файла, даже если названий элементов неизвестны. Есть также выражения для выяснения количества элементов и группах.

Упорядочить группы (Reordering Groups) Сдвиг, Сортировка, Перемешивание групп и т. д..

Быстро получить информацию Быстро выяснить, какие группы имеют определенные свойства с выражениями 'Выполнить вычисления (Perform Calculations)'.

Управляющие символы (Escape Characters) Управляющие символы могут быть использованы в названиях групп, названиях элементов и значениях элементов. Это позволяет использовать новые строки, которые будут представлены например в данных.

И ещё множество новых функций...

К началу

Негативные аспекты

Ини ++ имеет побочный эффект который может быть как положительным так и отрицательным. Ini++ загружает весь Ini в память, а затем сохраняет всё обратно на диск в наиболее эффективном виде, при сохранении. Это означает что он не сохраняет структуру файла. Под этим подразумевается, что если вы открыли файл Ini в Ini++ и затем его снова сразу сохранили, вы потеряете следующее:

  • Комментарии (т.е. что-либо после точки с запятой на строке).
  • Порядок элементов и групп (всё будет в алфавитном порядке).
  • Пустые строки, дополнительные пробелы, и так далее.
  • Если группа была ранее разделена и был использован режим 'Соединить (merge)' повторяющиеся группы, группа будет находиться в одном месте.

Это приведёт к наименьшему возможному размеру файла. Однако, если файл Ini будет регулярно редактироваться людьми, то это будет нежелательным. Чтобы обойти эту проблему, вы можете открыть файл в 'Режим только чтение (Read Only mode)'.

В качестве примера этого, рассмотрим следующий Ini файл:

Example Ini 2: The original file

;This group represents the scores for the various players
[Scores]
Jack = 32
Fred = 12
Oliver = 33

[Time]
Jack=15

;More scores
[Scores]
Alex = 22

Теперь, если этот файл будет загружен с настройками 'Повторяющиеся группы (Group repeat setting)' установить на соединение (см. раздел 2.3 для информации), и сохраним его снова, мы обнаружим что файл выглядит следующим образом:

Example Ini 3: Example 2 after being saved with Ini++

[Scores]
Alex=22
Fred=12
Jack=32
Oliver=33
[Time]
Jack=15

Таким образом, дополнительные пустые строки и пустое пространство были удалены. Регистр названий групп и элементов изменился. Произведено различное упорядочивание, комментарии были удалены и ранее разделённые группы были соединены.
