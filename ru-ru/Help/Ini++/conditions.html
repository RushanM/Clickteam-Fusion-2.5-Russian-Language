Основное меню (Main Menu)

Группа существует (Group Exists)

Проверяет, существует ли указанная группа. Если включено 'Разрешить пустые группы (Allow empty groups)', то группа может существовать, но не иметь элементов. Когда этот режим не выбран, то группа может существовать, даже если она не имеет элементов (но не каждая группа существует).

Группа имеет элемент (Group Has Item)

Проверяет, существует ли указанный элемент.

Элемент имеет хешированное значение (Item Has Hashed Value)

Проверяет если значение элемента равно заданной строке, хешировано. Третий параметр позволяет получить значение элемента по умолчанию. Если требуется это будет хешировано, так что нет никакой необходимости хешировать его!
Например, предположим что у нас есть файл Ini:

[group]
item=2063c1608d6e0baf80249c42e2be5804

(Значение элемента является значение хеш)

Теперь, если мы сравним пункт (в 'группе') к 'значению' это будет верно. Если мы сравним с его 'деревом' будет неверно, так как 'дерево' хеширует к чему-то еще.
Теперь предположим, что мы сравниваем некоторый элемент который не существует и даём ему значение по умолчанию 'значением' и сравниваем его с 'значением'. Это будет верно. Но если мы даем ему значение по умолчанию '2063c1608d6e0baf80249c42e2be5804' и сравним его со 'значением', то это будет неверно, даже если 'значение' хеширует к этому значению.

При соединении (On Merge)

Это событие вызывается, когда действие слияния было успешным. К ним относятся: Соединение из файла, соединение группы из файла, соединение из объекта и соединение группы из объекта.
'Успешный' означает, что либо файл либо объект существует.

При изменении данных (On Data Modified)

Это событие вызывается, если данные в Ini++ объекте были изменены каким-либо образом. Для глобальных данных, это происходит даже в том случае, если данные из другого Ini++ объекта изменяются. Обратите внимание, что переключение глобального слота данных который использует объект, не учитывается как изменение данных (так как никакие изменения не происходят).

Любое действие, которое изменяет данные вызывает это событие. Поэтому если действие которое изменило данные было использовано из этого условия, будет происходить бесконечный цикл. Поэтому Ini++ объект запрещает этому происходить. Все действия, которые изменяют данные отключены из этого условия. Когда программа запущена из MMF, окно с сообщением предупреждает пользователя, что это произошло.

При новом элементе (On New Item)

Это событие вызывается, когда новый элемент добавляется в Ini++ объект.

При сохранении (On Save)

Это событие вызывается, когда данные сохраняются. Это включает в себя автоматическое сохранение.

Текущая группа (Current Group)

В этом разделе приведены условия, аналогичные 'Группа существует (Group Exists)', 'Группа имеет элемент (Group Has Item)' и 'Элемент имеет хешированное значение (Item Has Hashed Value)' вышеперечисленным. Отличие состоит в том что группа относится к текущей группе.

К началу

Диалоговое окно (Dialog Box)

При открытии (On Open)

Это событие вызывается, когда диалоговое окно открыто.

При закрытии (On Close)

Это событие вызывается при закрытии диалогового окна. Это не включает в себя, когда оно закрыто из-за смены фрейма или завершения приложения.

При изменении (On Modified)

Это событие вызывается, когда Ini++ объект изменяется через диалоговое окно. 'При изменении данных (On Data Modified)' будет также верно и в этой ситуации.

К началу

Результаты поиска (Search Results)

При найденных результатах (On Results Found)

Вызывается, когда был поиск и есть результаты поиска. (В этом случае поиски 'and', 'or' и 'xor', это означает что подсчет результатов не равен нулю.)

При не найденных результатах (On No Results Found)

Вызывается, когда был поиск и нет результатов поиска. (В этом случае поиски 'and', 'or' и 'xor', это означает что подсчет результатов равен нулю.)

К началу

Дополнительные функции (Extra Functions)

Эта маска совпадает (Is Wildcat Match)

Проверяет соответствует ли строка шаблону точно так же, как 'Выполнить поиск (Perform Search)'. Это позволяет использовать '.', '*' и '!'.
